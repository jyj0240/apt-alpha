# APT Alpha 개편 설계서 (v1)

> **목표**: 헤지펀드급 분석 엔진은 그대로 두되, "전문가 도구"의 얼굴을
> **일반 실수요·투자자가 쓰는 앱**의 얼굴로 바꾼다.
> 사용자에게 *데이터와 해석 숙제*가 아니라 *한 문장의 답*을 준다.

- 대상 사용자: **일반 실수요·투자자 우선** (전문 지표는 'Pro'로 숨김)
- 개편 강도: **과감한 재구성 (11페이지 → 5)**
- 이 문서 범위: 정보구조 / 화면 설계 / 용어 / 마이그레이션 청사진 (코드 변경 전 단계)

---

## 1. 설계 원칙

| 원칙 | 의미 | 지금과의 차이 |
|---|---|---|
| **답 먼저** | 화면 최상단에 한 문장 판정을 둔다 | 지금은 KPI·탭·표를 던지고 해석은 사용자 몫 |
| **점진 공개** | 핵심 1~2개만 보이고, 나머지는 "자세히"로 접는다 | 지금은 모든 층이 펼쳐져 압도됨 |
| **사람 말** | 알파·베타·R²·MDD를 일상어로 번역 | 지금은 CFA 교재 용어가 날것 |
| **마찰 최소** | 기본값으로 바로 진입, 선택은 검색 1줄로 | 지금은 4단계 온보딩 + 페이지마다 셀렉트박스 반복 |
| **일관성** | 입력은 사이드바 하나, 카드 레이아웃 통일 | 지금은 홈(pills)과 내부(사이드바) 두 시스템 |

### "답 먼저" 3층 구조 (모든 페이지 공통 틀)

```
┌──────────────────────────────────────────────┐
│ 1층  판정 카드 — 한 문장 + 신호등(🔥/⚖️/❄️)      │  ← 누구나 3초 안에 이해
├──────────────────────────────────────────────┤
│ 2층  핵심 숫자 1~2개  +  대표 차트 1개            │  ← 근거 (자명한 시각화)
├──────────────────────────────────────────────┤
│ ▸ 자세히 보기 (expander)                        │  ← 표, 동별 비교 등
│ ▸ 전문가 지표 (expander, Pro 모드에서만)         │  ← 베타/R²/변동성 원시값
└──────────────────────────────────────────────┘
```

## 2. 정보구조: 11페이지 → 5

### 통합 매핑

| 신규 페이지 | 답하는 질문 | 흡수하는 기존 페이지 |
|---|---|---|
| 🏠 **오늘의 시장** | "내 관심 지역, 지금 분위기 어때?" | (신규 대시보드) + 1·일부 요약 |
| 🗺️ **동네 보기** | "이 동네, 사도 될 타이밍이야?" | 1 가격추이 · 2 지도 · 4 전월세 |
| 🏢 **단지 보기** | "이 단지 이 가격, 비싼 거야 싼 거야?" | 5 단지분석 · 3 예측 · 6 단지랭킹 · 11 비교 |
| 💰 **내 자금** | "얼마 빌리고 매달 얼마 내?" | 7 대출 시뮬레이터 |
| ⚙️ **전문가 분석** (Pro) | "알파/베타로 뜯어보고 싶다" | 8 심층 · 9 알파 · 10 스크리닝 |

→ 첫 화면 인지 부하 **11 → 4(+Pro 1)**. 어려운 기능은 *삭제가 아니라 격리*.

### 메뉴 구성 (사이드바 네비게이션, 섹션 분리)

```
일상
  🏠 오늘의 시장      (기본 진입)
  🗺️ 동네 보기
  🏢 단지 보기
  💰 내 자금
─────────────────
전문가용  ⓘ Pro
  ⚙️ 전문가 분석      (알파·베타·스크리닝·시계열 분해)
```

## 3. 페이지별 상세 설계

### 🏠 오늘의 시장 (홈 / 대시보드)

**역할**: 진입 즉시 "관심 지역 현황"을 한눈에. 온보딩을 강요하지 않고, 기본값(예: 강남구·최근 1년)으로 바로 무언가를 보여준다.

```
┌──────────────────────────────────────────────┐
│ 강남구 · 최근 1년               [지역·기간 변경 ▾]│
├──────────────────────────────────────────────┤
│ 🔥 지금 강남구는 "과열" 구간                      │
│ 최근 6개월 +3.2%, 서울 평균(+1.1%)보다 빠릅니다.   │
├───────────┬───────────┬───────────┬──────────┤
│ 평당 8,900 │ 6개월 +3.2%│ 거래 1,240건│서울대비+58%│
│  만원     │  (상승세)  │  (활발)    │ (프리미엄) │
├──────────────────────────────────────────────┤
│  [ 가격 추이 미니 차트 ]                          │
├──────────────────────────────────────────────┤
│ 👉 지금 눈여겨볼 단지 (Top 3)                     │
│  1. 래미안OO  2. OO자이  3. OO파크   [더 보기]    │
├──────────────────────────────────────────────┤
│ 바로가기:  🗺️ 동네 보기   🏢 단지 보기   💰 내 자금 │
└──────────────────────────────────────────────┘
```
- 판정 로직: 기존 `calc_period_change` + `calc_recent_momentum` + `calc_relative_premium` + `generate_insight_text` 재활용 → 신호등(🔥과열 / ⚖️중립 / ❄️침체) 규칙으로 매핑.
- Top 3: `screen_complexes` 결과 상위 3 (용어는 "눈여겨볼 단지"로, 점수 내부 처리).
- 온보딩 4단계는 **선택**으로. 스킵 시 기본값 진입, 상단 "변경" 버튼으로 호출.

### 🗺️ 동네 보기 (= 가격추이 + 지도 + 전월세)

**답**: "이 동네, 사도 될 타이밍이야?"

```
┌──────────────────────────────────────────────┐
│ 강남구                          [지역·기간 변경 ▾]│
├──────────────────────────────────────────────┤
│ ⚖️ 강남구는 "상승 후 숨고르기" 구간               │
├──────────────────────────────────────────────┤
│ [ 가격 추이 ][ 거래량 ][ 지도 ][ 전월세 ]  ← 탭   │
│  · 가격 추이: 매매가/평당가 라인 (자명한 주석)     │
│  · 지도: 구별 평당가 choropleth                   │
│  · 전월세: 전세가율 추이 + 전세/월세 비중          │
├──────────────────────────────────────────────┤
│ ▸ 동별 비교 (자세히)                              │
│ ▸ 전문가 지표 (Pro): 변동성 히트맵, 동별 격차표    │
└──────────────────────────────────────────────┘
```
- 데이터: `aggregate_by_gu_month`, `aggregate_by_dong_month`, `price_trend_chart`, `price_per_sqm_chart`, `trade_volume_chart`, `map_view`, `calc_jeonse_ratio`, `calc_contract_type_ratio`.
- 현재 1번 페이지의 "랭킹/히트맵" 탭과 "변동성/동별격차" 표 → Pro expander로 강등.
- "읽는 법" 문단 제거 → 차트에 직접 주석(마지막 점 라벨, 기준선) 부착.

### 🏢 단지 보기 (= 단지분석 + 예측 + 랭킹 + 비교)

**답**: "이 단지 이 가격, 비싼 거야 싼 거야?" — **1단지 = 1답**

```
┌──────────────────────────────────────────────┐
│ [ 🔍 단지 검색: "강남 래미안..." ]                │
├──────────────────────────────────────────────┤
│ 래미안OO · 84㎡                                  │
│ ❄️ 고점 대비 -12%, 같은 동네 평균보다 8% 쌉니다.   │
│ 예상 적정가 13.2억 ~ 14.0억 (현 호가 13.5억)      │
├───────────┬───────────┬──────────────────────┤
│ 최근 13.5억│ 고점 -12% │ 적정가 대비 -3% (저평가)│
├──────────────────────────────────────────────┤
│ [ 실거래 추이 + 고점/예측밴드 차트 ]              │
├──────────────────────────────────────────────┤
│ 🆚 비슷한 단지 비교 (자동 추천 3곳)               │
│ ▸ 자세히: 거래 내역, 면적별                       │
│ ▸ 전문가 지표 (Pro): 유동성 점수, 최대낙폭, 베타   │
└──────────────────────────────────────────────┘
```
- 데이터: `get_apt_price_history`, `calc_apt_peak_drop`, `predictor.AptPricePredictor`(적정가 밴드), `find_comparable_complexes`(비슷한 단지), `calc_investment_score`(내부), `calc_max_drawdown`/`calc_liquidity_score`(Pro).
- 4개 페이지의 구·동·단지 셀렉트박스 반복 → **검색 1줄**로 통일.
- "랭킹"은 단지 보기 진입 전 "이 동네 단지 줄세우기" 보조 뷰로 흡수(또는 동네 보기 내 탭).

### 💰 내 자금 (= 대출 시뮬레이터)

**답**: "얼마 빌리고 매달 얼마 내?" — 거의 그대로 유지하되 답 먼저로.

```
┌──────────────────────────────────────────────┐
│ 매매가 [13.5억]  연소득 [8천만]  기존대출 [없음] │
├──────────────────────────────────────────────┤
│ ✅ 최대 X.X억까지 가능, 월 상환 OOO만원           │
│ (DSR 40% 기준, 30년 원리금균등)                  │
├──────────────────────────────────────────────┤
│ [ 상환 시뮬레이션 차트 ]                          │
│ ▸ 자세히: 금리·기간별 민감도                       │
└──────────────────────────────────────────────┘
```

### ⚙️ 전문가 분석 (Pro) — 알파·베타·스크리닝·시계열

**역할**: 기존 8·9·10번을 **거의 그대로** 보존 (이 사용자층은 용어를 안다). 단, 일상 메뉴와 시각적으로 분리하고 진입 시 "전문가용" 배지 표시.

- 알파/베타 분해 (9번) · 다중 팩터 스크리닝 (10번) · 시계열 분해/변동성/MDD (8번)
- 일반 페이지의 Pro expander와 이 섹션은 **같은 함수**를 공유 → 중복 최소화.

## 4. 공통 컴포넌트 (신규 / `design_system.py`에 추가 권장)

| 컴포넌트 | 역할 | 비고 |
|---|---|---|
| `verdict_card(signal, headline, sub)` | 1층 판정 카드 (🔥/⚖️/❄️ + 문장) | 모든 페이지 재사용 |
| `signal_from_metrics(...)` | 지표 → 신호등 규칙 매핑 | 과열/중립/침체 임계값 한 곳에서 관리 |
| `hero_metrics(items)` | 핵심 숫자 1~4개 일관 렌더 | 라벨은 사람 말 |
| `pro_section(...)` | Pro 모드에서만 펼치는 expander | 모드 토글과 연동 |
| `complex_search()` | 검색형 단지 선택 (구·동·단지 통합) | 셀렉트박스 4개 대체 |

## 5. 용어 번역 사전 (라벨만 교체, 내부 계산은 불변)

| 코드/현재 | 사용자 라벨 | 한 줄 설명(툴팁) |
|---|---|---|
| 알파 (alpha) | **단지 고유 경쟁력** | 시장을 빼고 이 단지만의 힘 |
| 베타 (beta) | **시장 민감도** | 시장이 흔들릴 때 얼마나 따라 움직이나 |
| R² | (일상: 숨김 / Pro: **시장 설명력**) | 가격이 시장으로 설명되는 정도 |
| 모멘텀 | **최근 상승세** | 최근 N개월 오른 정도 |
| 변동성 (volatility) | **가격 출렁임** | 가격이 얼마나 들쭉날쭉한지 |
| 최대낙폭 (MDD) | **고점 대비 최대 하락** | 고점에서 가장 많이 빠졌던 폭 |
| Value 팩터 | **가격 매력도** | 동급 대비 싼 편인지 |
| Liquidity | **거래 활발도** | 팔고 싶을 때 잘 팔리나 |
| Safety | **안정성** | 하방 리스크가 낮은지 |
| 프리미엄 | **서울 평균 대비** | 서울 전체보다 비싼/싼 정도 |
| 전세가율 | **전세가율** (유지) | 전세금이 매매가의 몇 % |

## 6. 필터 · 네비게이션 · 온보딩

- **입력 일원화**: 사이드바 `render_sidebar_filters()` 하나로. 홈의 4단계 pills 온보딩은 "처음 한 번" 또는 "변경" 버튼 호출형으로 축소.
- **기본값 진입**: 미설정 시 `강남구 · 최근 1년 · 전체 면적/연식`으로 바로 대시보드.
- **검색형 단지 선택**: 구→동→단지→면적 4단 셀렉트 → "강남 래미안 84" 한 줄 검색 + 자동완성.
- **session_state 계약 유지**: `selected_gus / start_ym / end_ym / selected_area / selected_build_year` + 신규 `pro_mode`(bool), `selected_complex`.

## 7. 기술 구현 노트 (Streamlit)

- **네비게이션**: 현재 `pages/` 자동 네비 → `st.navigation`(섹션 그룹 지원)으로 전환하여 "일상 / 전문가용" 헤더 분리. `app.py`에서 페이지 객체를 그룹으로 등록.
- **Pro 모드**: 사이드바 토글(`st.toggle("전문가 모드")`) → `st.session_state["pro_mode"]`. 일반 페이지의 `pro_section()`과 전문가 페이지 노출을 동시에 제어.
- **중복 제거**: 4개 페이지에 흩어진 동일 데이터 로딩(`get_data` + `clean_*` + `filter_*`)을 공용 헬퍼(예: `data_pipeline.py`)로 추출 → 신규 5페이지가 공유.
- **차트 자명화**: "읽는 법" 문단 대신 `design_system`의 `add_last_point_label`/`add_reference_line`으로 차트 내 주석. 모바일은 기존 `show_chart` config 유지.
- **백엔드 무변경**: `data_processor / factor_model / screener / predictor`는 그대로. 이번 개편은 **표현 계층 한정**.

## 8. 마이그레이션 순서 (코딩 단계 — 승인 후 진행)

| 단계 | 작업 | 산출물 | 위험 |
|---|---|---|---|
| M1 | 공용 컴포넌트 + 데이터 파이프라인 헬퍼 추가 | `design_system` 확장, `data_pipeline.py` | 낮음 |
| M2 | `st.navigation` 도입 + Pro 토글, 메뉴 2섹션화 | `app.py` 라우팅 | 중간 |
| M3 | 🏠 오늘의 시장 (신규 대시보드) | `pages/홈` | 중간 |
| M4 | 🗺️ 동네 보기 (1·2·4 통합) | 신규 페이지 + 구 페이지 제거 | 중간 |
| M5 | 🏢 단지 보기 (5·3·6·11 통합) + 검색형 선택 | 신규 페이지 | 높음 |
| M6 | 💰 내 자금 (7 정리) / ⚙️ 전문가(8·9·10 이동) | 정리·배치 | 낮음 |
| M7 | 용어 번역 일괄 적용 + "읽는 법" 제거·주석화 | 전 페이지 | 낮음 |

각 단계는 독립 커밋. M2 이후에도 기존 페이지를 한동안 병행 유지해 회귀 위험을 줄인다.

## 9. 열린 결정 사항 (진행 중 확인 필요)

1. **적정가 밴드**: `predictor`(GradientBoosting) 출력을 그대로 "적정가"로 부를지, 신뢰구간/면책 문구를 어떻게 둘지.
2. **신호등 임계값**: 과열/중립/침체를 가르는 상승률·모멘텀·프리미엄 컷오프 수치 (초안 후 튜닝).
3. **랭킹의 위치**: 단지 줄세우기를 "단지 보기" 내부 탭으로 둘지, "동네 보기"에 둘지.
4. **기본 진입 지역**: 강남구 고정 vs 마지막 선택 기억(`session_state` 지속).
5. **Pro 모드 기본값**: 끄기(일반 우선) 확정.
